Node.js


by Kazi Jamal and Ahmed Sultan, pd 9

What is Node.js?

Node.js is a JavaScript runtime platform, based off Google Chrome's V8 JavaScript engine

It is...

  • OPEN SOURCE
  • SCALABLE
  • SUPPORTED ACROSS PLATFORMS

What are Node.js' uses?

There are two main ways people today use Node.js:

  1. Creating utilities to be used locally, or on servers
  2. Creating web servers and applications
    • Node.js attempts to optimize processing speeds and scalability; strong choice for the development of web applications
    • Cornerstone of some modern-day 'web development stacks' (MEAN, MERN, etc.)
    • Lots of big-name corporations have shifted toward Node.js (Microsoft, Netflix, LinkedIn, PayPal, Uber, etc.)

How do we use Node.js?

Unlike the JavaScript many of us have experienced in SoftDev—whose runtime is confined entirely within the browser—Node.js gives your JavaScript code an environment to run on your computer...

  • Put simply: with Node.js, you can run your JavaScript code in a terminal/locally

Okay... what's npm?

npm is a package manager for JavaScript—and it is the default package manager for Node.js

You can install packages from npm to use in your project

  • by using the command npm init, we can initialize a package which includes a package.json—which, in turn, stores dependencies (modules you have already installed) and other metadata about your project/package

Popular npm packages

  • express: "fast, unopinionated, minimalist web framework for Node.js"
  • lodash: utility library, adding methods to iterate through arrays, strings, etc.
  • react: a library for building user interfaces

Why use Node.js?

Advantages of Node.js

  • Allows programmers to write both front-end and back-end of web applications in a single language
  • Very large community—lots of learning resources and npm modules to utilize
  • High performance, with Google's V8 JavaScript engine
  • Can process many requests concurrently, better than Ruby or Python
  • Lightweight and scalable

Disadvantages of Node.js

  • Lacking in performance for heavy computational and CPU bound tasks
  • Lacking in maturity—the Node.js API is frequently updated and modified
  • Some npm modules lack thorough documentation

Node.js vs. Flask/Python

  • Python primarily use pip for library management... while Node.js uses npm
  • Python comes with a built-in environment manager... while Node.js requires you to install one (for example: nodeenv) separately.
  • Flask often has more mature and stable libraries with solid documentation... while Node.js modules sometimes lack thorough documentation.
